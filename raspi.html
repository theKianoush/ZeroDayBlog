Building a Portable Virtualization & Security Lab with Raspberry Pi and VMware ESXi

By: Kian Ranjbar

I built a small, portable virtualization and security lab using a Raspberry Pi running VMware ESXi. The goal was to create a hands-on environment where I could experiment with Linux, virtualization, networking, and cybersecurity concepts in a controlled setting.

Project Overview

The foundation of this project was a Raspberry Pi configured to run VMware ESXi, effectively turning it into a lightweight, portable virtual machine hub. This setup allowed me to deploy and manage multiple virtual machines for learning, testing, and experimentation—essentially a compact home lab that could be accessed remotely.

Linux & Virtualization Experience

Through this project, I became comfortable working with multiple Linux distributions, with a strong focus on:

  • Raspberry Pi OS (Raspbian) – My favorite overall due to its simplicity, stability, and ease of use.
  • Kali Linux – Particularly valuable for its preinstalled offensive and defensive security tools.

Running these systems as virtual machines helped solidify my understanding of Linux internals, package management, system services, and remote access.

Security Tools & Networking Concepts

I used this lab to explore practical cybersecurity tools and techniques, including:

  • Wireshark for network traffic monitoring and analysis
  • Nmap for reconnaissance and penetration testing
  • Nessus for vulnerability scanning
  • VPNs and Tor for understanding anonymity and IP obfuscation
  • ProxyChains to route traffic through multiple proxies

All testing was conducted in an isolated, ethical environment designed strictly for learning and experimentation.

System Hardening & Defensive Security

Beyond offensive tools, I focused heavily on system hardening and defensive practices:

  • SSH hardening (key-based authentication and non-standard ports)
  • PGP key usage for secure communication
  • Firewall configuration using UFW
  • Simulated DDoS scenarios to understand system behavior under stress

These exercises helped me better understand attack surfaces and the importance of layered security.

Web Development & Backend Experimentation

I also used the lab to experiment with classic, bare-metal web development:

  • Built a simple car rental database interface using PHP
  • Designed and managed the backend with MySQL
  • Deployed the application using Apache as the web server

This reinforced full-stack fundamentals and demonstrated how application-level decisions interact with system-level security.

Remote Access & Usability

One of the most valuable aspects of the project was remote accessibility:

  • Secure SSH tunneling into the Raspberry Pi
  • VNC-based remote desktop access to virtual machine frontends

Being able to securely manage and interact with the lab from anywhere significantly enhanced its practicality and reinforced real-world administration skills.

Key Takeaways

  • Solidified Linux administration skills
  • Gained practical experience with virtualization on limited hardware
  • Applied cybersecurity concepts in a realistic environment
  • Practiced web development, software testing, and ethical hacking fundamentals

Overall, this Raspberry Pi ESXi lab became a versatile learning platform and one of the most effective ways I’ve found to bridge theory with real-world technical execution.
